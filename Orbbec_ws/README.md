# Orbbec_ws 视觉侧说明

这个 README 面向视觉侧队友，说明 `Orbbec_ws` 里需要做的最小更新，目的是让相机端可以向机械臂侧 `robot_ws` 提供兼容的目标点数据。

## 目的

机械臂侧 `robot_ws` 正在重构为一个闭环、逐步执行的抓取系统。

你在 `Orbbec_ws` 的工作是：
- 输出一个稳定的视觉目标
- 让目标包含置信度信息
- 确保目标已转换到 `base_link` 坐标系
- 以机械臂侧能直接消费的格式发布

## 需要更新的内容

### 1. 统一视觉目标消息格式

视觉侧应该发布目标到一个机械臂侧可订阅的主题，例如：

- `/camera_target_base`

目标消息至少应包含：

- `x`, `y`, `z`（`base_link` 坐标）
- `confidence`（检测置信度）
- `timestamp`（目标生成时间）
- `frame_id = base_link`

推荐后续扩展字段：

- `target_id`
- `object_name`
- `is_stable`
- `estimated_gripper_width`

### 2. 确保目标已经在 `base_link`

机械臂侧假设视觉目标已经转换到了机械臂基坐标系。

因此，视觉侧应当：

- 直接发布 `base_link` 坐标，或
- 发布带有正确 TF 的带时间戳数据，并保证最终在 `base_link` 中使用

当前机械臂侧期望收到的 `frame_id` 是 `base_link`。

### 3. 加入置信度与稳定性判断

机械臂任务层会基于：

- 最新置信度
- 帧间目标漂移
- 目标稳定性
- 当前抓取阶段

来决定“继续、等待、重规划、恢复”等动作。

所以视觉侧应该：

- 为每个目标计算置信度
- 尽量避免发布噪声大、置信度低的目标
- 可以在目标稳定时额外发布稳定性标志

### 4. 保持视觉侧职责简单

目前视觉侧只负责：

- 检测目标物或抓取点
- 将目标转换到 `base_link`
- 发布目标消息
- （可选）以受控频率发布

不要在 `Orbbec_ws` 里去做路径规划、运动执行、抓取阶段管理等。

## 期望结果

在 `Orbbec_ws` 里完成一个小更新，满足以下条件：

- 明确一个目标发布主题，例如 `/camera_target_base`
- 发布 `PointStamped` 或类似的姿态消息，`frame_id = base_link`
- 包含 `confidence` 和 `timestamp`
- 保持与机械臂侧接口稳定，不随意改动机械臂侧协议

## 给机械臂侧的备注

机械臂侧正在重构成：
- 任务层只做闭环决策
- 执行层成为唯一真实硬件控制者
- 视觉侧只负责输出稳定目标

你输出的目标一定要稳定、坐标一致，机械臂侧会负责“单步执行 + 反馈后再决定下一步”。
